- pertanyaan dimasukin ke state ask
- semua response dari fsm
- bikin class baru buat nampung data barang
- di state order dicek, udah ada blm data barang, klo ada lgsg konfirmasi order: ya (berhasil), ngga (back to ask)
- pengecekan di db 

- di state order dicek udah login blm, klo blm lgsg ke state auth, klo udah proses insert data barang ke db

# TANYA STOK
(ask)
user: asus ada? -> extract entity merk laptop = asus
bot: tentu, berikut list tipe laptop dari asus
bot: [menampilkan list tipe dari laptop asus]
bot: tipe mana yang Anda inginkan?
user: [memilih tipe laptop]
bot: baik, untuk jumlahnya berapa kak?
user: 1

(order) -> pertama dicek dlu apakah sudah login menggunakan variabel is_login, klo udah proses ke insert data ke db, klo ngga ke state auth 
default: is_login = False -> jadi langsung masuk ke kondisi blm login jadi ke auth dlu

(auth)
bot: untuk melanjutkan transaksi silahkan login terlebih dahulu
bot: apakah Anda sudah memiliki akun?
user: udah

(login)
bot: masukkan email dan password anda, contoh: nilam@mail.com 12345
user: [email password]
bot: Selamat datang [name]! Kami senang anda menggunakan layanan Kami

(order) -> is_login = True
bot: [menampilkan rangkuman pesanan, berupa merk, tipe, harga, dan jumlah laptop]
bot: apakah pesanannya sudah sesuai?
user: udah 
bot: baik, pesanan berhasil diproses!
bot: Apakah anda ingin membeli laptop lain?
user: ngga

(exit)
bot: Terimakasih telah menggunakan layanan kami :)
